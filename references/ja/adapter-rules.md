# アダプタ規則

ツール用アダプタファイルは軽量な入口であるべきで、独立した知識体系になってはいけません。

## 目的

対応する各コーディングツールは、最小限のツール固有表現だけで同じ `.omnicontext/` データを読み込むべきです。

## 必須動作

各アダプタはツールに対して次を要求します。

1. まず `.omnicontext/INDEX.md` を読む
2. タスク内容または現在の作業領域から対象プロジェクトを特定する
3. 対象プロジェクトの `overview.md` と `handoff.md` を読む
4. ワークスペース共通の標準やアーキテクチャに関係する場合のみ `shared/` を読む
5. ユーザー設定が関係する場合のみ `personal/` を読む
6. 永続的な変化があったら `handoff.md`、`todo.md`、`decisions.md` を更新する

## アダプタの範囲

アダプタは `AGENTS.md` や `CLAUDE.md` のようなツール固有の約束を記述してもよいですが、次を再定義してはいけません。

- ディレクトリ構造
- 設定の意味
- 文書の責務
- 更新ルール

これらは OmniContext のプロトコルと共有リファレンスで定義すべきです。

## 現在対応している入口ファイル

- `tools/codex/AGENTS.md`
- `tools/claude-code/CLAUDE.md`
- `tools/trae/TRAE.md`
- `tools/qoder/QODER.md`

実際の対象ツールがあるときだけ、新しいアダプタを追加してください。
